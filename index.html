Sidebar navigation access by role
What each role sees in the left navigation
Finance and HR pages use a separate access check — checkSectionAccess() — not a simple role check. A user of any role can see these pages if they have a Finance or HR privilege granted to them, or if their department name matches "finance" / "hr & sops".
Privilege (cross-role access) system
Layered on top of the role system to grant individuals extra section access without changing their role

Access levels
viewRead-only access to the section
editFull read + write access to the section
How access is resolved (3-tier check)
1
Role check (god-mode): If the user's role is owner, ceo, or admin → always grant access
2
Department membership: If the user belongs to a department named "finance" or "hr & sops" → grant access + edit by default
3
Explicit privilege record: If a row exists in user_privileges for this user + workspace + section → grant at stored level
Who can grant privileges
manage_privileges permissionOnly owner, ceo, admin have this permission in the role matrix

Leave approval privilege
Leave approval is not part of the user_privileges table. It uses an inline check: approver must be owner, admin, manager, OR have hr_sops edit access via checkEditAccess("hr_sops").
